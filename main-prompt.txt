### **MASTER BUILD PROMPT: "Pathfinder" v0.1 - College Counseling Platform MVP**

You are a senior full-stack architect + DevOps + compliance engineer. Build **"Pathfinder" v0.1** — a production-ready MVP of an AI-driven college counseling platform for a pilot of 100 US high school students, grades 9-12.

**Core Principles:**
1. **Dockerized**: One command `./start.sh` brings up entire stack. `./cleanup.sh` tears it down
2. **Few, Optional Agents**: Max 2 background agents. All toggleable by admin with graceful fallback
3. **Role-Based**: Student, Parent, Counselor, Admin each get dedicated login + dashboard
4. **Compliance First**: FERPA/COPPA safe. Audit logs. No PII in git
5. **Ship in Phases**: Phase 1 = manual core. Phase 2 = enable agents. Phase 3 = advanced features

#### **1. TECH STACK - FINAL**

**Frontend**
- **Framework**: React 18 + Vite + TypeScript
- **UI**: MUI v5 with 3 themes: "Light Academic", "Dark Modern", "High Contrast". User toggleable
- **Routing**: React Router v6
- **Charts**: Vega-Lite via `react-vega` for ALL data viz. No Recharts
- **PDF**: `@react-pdf/renderer` + render Vega charts to PNG for export
- **State**: TanStack Query + Zustand
- **PWA**: `vite-plugin-pwa` for offline + Add to Home Screen

**Backend**
- **Framework**: Node.js 20 + Express + TypeScript
- **ORM**: Prisma with `provider = "mysql"`
- **Auth**: Lucia Auth with MySQL adapter. Email/password + Google OAuth. RBAC enforced
- **Queue**: BullMQ + Redis for 2 agents only
- **Email**: Resend with `react-email` templates
- **Files**: `multer-s3` to local S3-compatible bucket. Whitelist: pdf, docx, png, jpg

**Database**: MySQL 8.0 `utf8mb4`. No pgvector. JSON columns for flexible data.

**Infra**: Docker Compose. Services: `web`, `api`, `db`, `redis`, `adminer`. Healthchecks + `depends_on`.

#### **2. ROLES & DASHBOARDS**

| Role | Route | Core View | Permissions |
| --- | --- | --- | --- |
| **Student** | `/dashboard` | GPA trend, EC donut, Roadmap Gantt, Admit% gauges, Next 3 deadlines. Vega-Lite | CRUD own academics, ECs, colleges. Chat Co-Pilot |
| **Parent** | `/parent` | “College Ready %” gauge, GPA vs Target bar, Deadline timeline, “What Changed” log | Read-only by default. Comment if enabled by counselor |
| **Counselor** | `/counselor` | Student table sortable by risk, Avg GPA chart, Agent queue stats | CRUD any student, bulk CSV import, invite users, export PDF |
| **Admin** | `/admin/settings` | API Keys, Agent Controls, Feature Flags, Compliance, Audit Logs | Toggle agents, rotate keys, set data retention |

**Privacy**: Parent sees grades only if student age<18 OR student toggles “Share”. Enforced in API middleware + `ferpa_release` table. All access logged to `audit_logs`.

#### **3. FEATURES FOR v0.1**

**Phase 1 - Core Manual Loop:**
1. **Auth + Invite**: Counselor invites students. Student invites parent
2. **Academic Tracker**: Add courses/grades. “What-if” GPA simulator
3. **EC Logger**: Add club/hours/impact. Upload proof
4. **College List**: Search 4000+ colleges seeded from IPEDS. Show 25-75th stats. Manual admit% heuristic
5. **Roadmap Engine**: Rules-based 4-year plan. “If CS + 10th grade → suggest AP CS next year”
6. **Document Vault**: Upload transcript/essay. Versioned. 24h share links
7. **Co-Pilot Chat**: Inline LLM, not agent. Function calling: `get_gpa()`, `draft_email()`, `explain_term()`. Context = student JSON

**Phase 2 - Optional Agents:**
1. **ScoutAgent**: Scrapes web for local clubs via Serper.dev + cheerio. Admin toggle + concurrency 1-5. Fallback = manual add form
2. **ReportAgent**: Generates PDF with Vega charts. Admin toggle. Fallback = JSON export
3. **GuardianAgent**: Mandatory. Runs before any LLM/scrape. Blocks PII, essay writing, self-harm. Not toggleable

**Phase 3 - Counselor Tools:**
1. **LoR Manager**: Request → auto-email teacher → track → reminders D-14/D-7/D-3
2. **Scholarship Matcher**: 5000+ scholarships. Rules engine. “$ earned” Vega funnel chart
3. **College Visit Tracker**: Log visits → updates demonstrated interest score
4. **Interview Prep**: Co-Pilot mock interview with audio record + feedback
5. **Notifications**: Weekly digest, D-7 deadlines. Email + in-app. Parent opt-in

#### **4. ADMIN-CONTROLLED AGENTS**

Only 2 agents allowed in v0.1. All others must be inline code.

**`/admin/settings` → Agent Controls tab:**
```
[ ] Enable Opportunity Scout [Test Run] Concurrency: 2
    Finds local clubs. Disable to add manually.
[x] Enable Report Generator [Test Run] Concurrency: 1
    Builds PDFs. Disable to save server time.

GuardianAgent: Always On [View Logs]
```

If disabled, UI hides feature + shows helper text. Code checks `config.agent.scout.enabled` before queueing job.

#### **5. DATABASE SCHEMA - KEY TABLES**

```prisma
model User {
  id Int @id @default(autoincrement())
  email String @unique
  role String // student|parent|counselor|admin
  name String
  passwordHash String?
  student Student?
  parentLinks StudentParent[]
  counselorStudents Student[] @relation("CounselorStudents")
}

model Student {
  id Int @id @default(autoincrement())
  userId Int @unique
  user User @relation(fields: [userId], references: [id])
  grade Int
  zipCode String
  counselorId Int?
  counselor User? @relation("CounselorStudents", fields: [counselorId], references: [id])
  parents StudentParent[]
  shareWithParents Boolean @default(true) // FERPA toggle
}

model StudentParent {
  studentId Int
  parentId Int
  permissions Json @default("{\"viewGrades\":true,\"viewRoadmap\":true,\"comment\":false}")
  @@id([studentId, parentId])
}

model SystemConfig {
  id Int @id @default(1)
  openaiApiKey String? @db.Text // AES-256 encrypted
  serperApiKey String? @db.Text // AES-256 encrypted
  encryptionKey String // from env, not UI
  agentConfig Json @default("{\"scout\":{\"enabled\":false,\"concurrency\":2},\"report\":{\"enabled\":false,\"concurrency\":1}}")
  featureFlags Json @default("{\"scholarships\":true,\"interviewPrep\":false}")
  dataRetentionMonths Int @default(12)
  updatedAt DateTime @updatedAt
  updatedBy Int?
}

model AgentLog {
  id Int @id @default(autoincrement())
  agentName String // scout|report|guardian
  studentId Int?
  status String // queued|running|success|error
  input Json?
  output Json?
  error String? @db.Text
  durationMs Int?
  createdAt DateTime @default(now())
}

model AuditLog {
  id Int @id @default(autoincrement())
  userId Int
  action String // "VIEW_GRADES", "UPDATE_API_KEY"
  targetType String // "Student", "SystemConfig"
  targetId String?
  metadata Json?
  ip String?
  createdAt DateTime @default(now())
}

// + Academic, Extracurricular, College, Application, Document, RecommendationRequest, Scholarship, CollegeVisit tables
```

#### **6. DOCKER + SCRIPTS**

**`docker-compose.yml`**: `web`, `api`, `db`, `redis`, `adminer`. Volumes: `mysqldata`, `redisdata`. Healthchecks for db + redis.

**`start.sh`:**
1. Check Docker running
2. Create `.env` from `.env.example` if missing. Auto-generate `JWT_SECRET` + `ENCRYPTION_KEY` with `openssl rand -base64 32`
3. `docker compose build && docker compose up -d`
4. Wait for `db` healthy via `mysqladmin ping`
5. Echo URLs: Web http://localhost:3000, API :4000, DB UI :8080
6. Tail logs

**`cleanup.sh`:** `docker compose down -v && docker system prune -f`

**`.gitignore`**: Must ignore `.env`, `node_modules`, `dist`, `mysqldata`, `*.log`, `.DS_Store`. Never commit secrets.

**`.env.example`:**
```env
DATABASE_URL="mysql://root:pathfinder@db:3306/pathfinder"
REDIS_URL="redis://redis:6379"
JWT_SECRET=
ENCRYPTION_KEY=
OPENAI_API_KEY=

VITE_API_URL=http://localhost:4000
```

#### **7. SECURITY + COMPLIANCE**
1. **COPPA**: Block sign-up if DOB <13 years ago
2. **FERPA**: `shareWithParents` check on every grade/EC API call. 403 if false
3. **Encryption**: All API keys AES-256-GCM encrypted in DB using `ENCRYPTION_KEY` from env
4. **Audit**: Log all reads of PII + all admin config changes
5. **Rate Limit**: 100 req/15min per user. Agent routes 10/hour
6. **Guardian**: Block prompts asking to write essays, fake awards, or self-harm content
